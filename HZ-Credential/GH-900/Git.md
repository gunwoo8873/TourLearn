# VCS(Version Control System)
파일 변경 내용을 추적하는 프로그램 또는 번들 형태로, 개별 파일 또는 전체 프로젝트의 이전 버전을 쉽게 복구하는 것. 다수 인언의 작업 영역에서 영향을 주지 않고 동시에 하나의 프로젝트 혹은 동일 파일에서 작업할 수 있도록 하는 것이 최종 목표이다.

* VSC는 다음과 같이 수행이 가능하다.
    * 프고르 프로젝트의 모든 변경 내용, 변경된 타임라인 및 작업자를 확인
    * 변경 내용과 설명을 확인하고자 하는 메시지를 포함
    * 전체 프로젝트 또는 개별 파일의 이전 버전을 검색
    * 실험적으로 변경할 수 있는 분기(Branch)를 생성하면, 이는 기본 분기에 영향을 주지 않고 여러 사람이 동시에 다양한 작업을 수행할 수 있다. 향후 유지할 변경 내용을 기본 분기에서 병합(Marge)할 수 있음
    * 새 릴리즈를 표시를 위해 버전에 태그 부여

# CVS, SVN(Subersion), Perforce
이전 VSC 인스턴스는 중앙 집중식 서버를 사용하여 프로젝트 기록을 저장, 중앙 집중식 구성에서는 하나의 서버가 잠재적으로 단일 실패 지점일 가능성이 높기에 분상형인 `Git`으로 프로젝트의 전체 기록이 클라이언트와 서버에 모두 저장되어 네트워크에 연결하지 않고 파일을 편집하여 로컬로 체크 인할 수 있기에 서버가 다운되더라도 복사본이 존재한다.

## Git 용어
* 작업 트리 : 
    * 작업 중인 프로젝트를 포함하는 중첩된 디렉터리 및 파일 트리
* 리포지토리 : 
    * 작업 트리의 최상위 수준에 있는 디렉터리로, 프로젝트의 모든 기록 및 메타데이터를 여기에 보관한다. 작업 트리에 포함되지 않는 리포지토리로, 공유 및 백업에 사용되고 일반적으로 흔히 `.git`이라는 하나의 디렉터리다.
* 해시 : 
    * 해시 함수에서 생성된 숫자로, 파일 또는 다른 개체의 콘텐츠를 고정 자릿수로 나타낸다. Git은 160비트 길이의 해시를 사용하여, 결과를 이전 해시와 비교하여 파일이 변경되었는지 여부를 확인할 수 있다. 파일 시간 및 날짜 데이터 스탬프가 변경되었지만 파일 해시가 변경되지 않은 경우는 파일 내용이 변경되지 않았다는 것.
