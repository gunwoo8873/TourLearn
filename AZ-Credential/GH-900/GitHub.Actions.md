# GitHub Actions

## 필드 구성
```yml
name: <Actions field file Name>

on:
    branches: <Checkout branche triger>


jobs:
    <Job actions field name>:
        runs-on: <Select action runtime OS>
        steps:
            - name: <Action Name>

<...>
```

## 가격 책정
리포지토리 내에서 직업 워크플로를 자동화할 수 있으며, 가격 책정 모델은 리포지토리 표시 유형 및 계정 유형에 따라 달라진다.

* Public
  * 공룡 리포지토리에 대해 무료로 제공되며 GitHub 호스팅 실행기에서 무제한 시간을 제공한다.
* Private
  * 각 계정은 계획에 따라 GitHub 호스팅 실행기를 위한 주어진 무료 시간과 스토리지를 제공하여 플랜에 해당하는 제한적인 사용 범위가 존재한다.

* 무료
  * 공용 리포지토리는 무료 시간을 무한하게 제공
  * 프라이빗 리포지토리는 플랜에 따라 일정 할당된 시간을 부여
    * Free : 2,000/month
    * Pro : 3,000/month
    * Team : 3,000/month
    * Enterprise : 50,000/month
* Runner 유형당 가격
  * Linux : $0.008/min
  * Windows : $0.016/min
  * MacOS : $0.08/min

## 측정된 사용량 보고서
GitHub는 요금제 제품의 사용량을 추적하기 위한 자세한 청구 및 소비 보고서를 제공한다. 이는 보고서를 통해 관리자는 비용을 모니터링하고, 리소스를 효율적으로 할당하며 조직 정책을 준수할 수 있다. 가상 머신에서 워크플로가 실행되는 CI/CD 자동화 도구이며, 이러한 워크플로에서 소요되는 시간은 리포시토리 유형, 실행기 유형 및 사용량에 따라 측정된다.

### 소비 추적
* GitHub 조직 또는 계정에서 설정 -> 청구로 이동한다.
* GitHub Actions 섹션에서 사용된 시간을 확인할 수 있다.
* 사용량은 리포지토리, 실행기 유형 및 남은 할당량에 따라 세분화된다.

### 최적화 전략
* 대용량 워크플로에 자체 호스트 실행기를 사용하여 비용 절감
* 종속성을 캐싱하고 중복 작업을 줄여 워크플로 스크립트를 최적화
* 필요한 경우 워크플로가 트리거되도록 제한(특정 브런치에서만 푸시)
