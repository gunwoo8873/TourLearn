# GitHub Clients

## 컴퓨터 및 주변 장치에서 라이선스 사용량 통계

### 컴퓨터 계정 및 주변 장치 서비스 이해
자동화, 스크립트 실행 또는 타사 도구와 통합하는 데 사용되는 GitHub 계정이다. 사용자와 독립적으로 작동하며 CI/CD에서 자주 사용되기에 각 컴퓨터 계정은 표준 사용자와 같은 GitHub 라이선스를 사용한다.

### 주변 장치 서비스
주변 장치 서비스는 API 요청을 통해 GitHub와 상호 작용하는 외부 통합이다.
그렇다면 추적을 왜 하는가에 대한 이유는 다음과 같다.

* 사용되지 않거나 과도한 라이선스를 식별
* 비용을 최적화하고 불필요한 지출을 방지
* 비활성 또는 잘못 구성된 자동화 계정의 보안 리스크를 모니터링

### 컴퓨터 계정 및 주변 서비스 라이선스 관리 모범 사례
* 정기적 계정 감사
  * 시간에 지남에 따라 조직은 리포지토리 및 시스템에 계속 액세스할 수 있는 사용되지 않거나 부실한 계정을 누적한다. 이는 비활성 계정의 손상으로 악용 리스크가 발생하여 보안 위험을 초래할 수 있기에 정기적인 감사는 활성 및 필요한 계정만 존재하도록 하여 무단 액세스에 대한 노출을 줄인다.
* API 사용량 모니터링
  * 많은 타사 애플리케이션, CI/CD 파이프라인 및 통합은 GitHub API 리소스 및 엔터프라이즈 라이선스를 사용한다 과더한 API 호출은 속도 제한으로 이어져 개발자의 생산성에 영향을 미칠 수 있기에 무단 혹은 식별할 수 없는 API 사용은 중요한 데이터 및 보안 취약성을 노출할 수 있다.
* 실행기 사용 최적화
  * 자체 호스팅 및 GitHub 호스팅 실행기는 CI/CD 워크플로를 실행하여 비효율적인 사용으로 불필요한 비용이 발생하는 사례가 존재 유휴 상태의 자체 호스팅 실행기는 컴퓨팅 리소스를 낭비하고 모니터링되지 않은 상태로 두면 조직이 보안 위험에 노출된다. 또한 사용한 만큼 지불하는 방식으로 운영되기에 최적화를 지속적으로 개선하지 않으면 비용을 줄일 수 없다는 문제가 존재한다.
* 사용자 계정 제한
  * 사용자 계정에는 리포지토리에 대한 불필요한 액세스 권한이 없어야 하므로 권한 상승의 위험이 줄어든다 손상된 경우 악용하여 소스 코드를 조작하거나, 악의적인 변경 내용을 배포하거나, 민감한 보안 내용들을 노출할 수 있기에 보안 정책을 적용하여 규정 준수를 보장하고 잠재적인 위반을 최소화할 수 있다.


## CLI

## Moile
장소와 환경에 제약없이 빠르게 작업을 수행할 수 있는 방법을 제공한다 신뢰할 수 있는 자체 클라이언트 애플리케이션을 통해 데이터에 액세스하는 안전한 방법이며 IOS와 Android 디바이스에 사용이 가능하다. 다음과 같이 여러 작업을 수행이 가능하다.
    
* 스레드나 기타 알림을 음소거 하거나 혹은 확인 및 필터링 하여 github.com 알림을 관리 혹은 심사 및 삭제가 가능하다.
* 이슈와 끌어오기 요청을 읽고 협업을 요청한다.
* 사용자, 리포지토리, 조직을 검색하고 찾아볼 수 있어 상호 작용을 가능
* 외부 사용자 이름을 언급할 때 알림을 수신
* 특정 사용자 지정 시간에 대한 푸시 알림 예약
* 2단계 인증을 사용하여 계정 보호
* 인식할 수 없는 디바이스에서 로그인 시도 확인

## Desktop
Git 워크플로를 간소화하는 데 도움이 되는 오픈 소스 애플리케이션이다. 사용자와 팀 간의 협업을 용이하게 하고 팀 내에서 Git 및 GitHub 모범 사례를 공유할 수 있으며 Windows와 MacOS에서 사용 가능하다. 다음과 같이 여러 수행 가능한 작업이 존재한다.

* 리포지토리를 추가하여 기존 프로젝트를 로컬로 관리
* GitHub에서 리포지토리를 복제하여 신속하게 로컬로 프로젝트를 설정하고 작업 시작
* 커밋에 변경 내용을 대화형으로 추가하면 커밋하기 전에 변경 내용을 보다 쉽게 관리하고 검토할 수 있다.
* 공동 작성자를 커밋에 빠르게 추가하여 공동 작업 프로젝트에서 적절한 특성을 보장한다.
* CI 상태를 확인하여 코드 품질을 확인하고 공동 작업을 간소화한다.
* 변경된 이미지를 비교하여 이미지 파일의 업데이트를 시각적으로 검토
