# VCS(Version Control System)
파일 변경 내용을 추적하는 프로그램 또는 번들 형태로, 개별 파일 또는 전체 프로젝트의 이전 버전을 쉽게 복구하는 것. 다수 인언의 작업 영역에서 영향을 주지 않고 동시에 하나의 프로젝트 혹은 동일 파일에서 작업할 수 있도록 하는 것이 최종 목표이다.

* VSC는 다음과 같이 수행이 가능하다.
    * 프고르 프로젝트의 모든 변경 내용, 변경된 타임라인 및 작업자를 확인
    * 변경 내용과 설명을 확인하고자 하는 메시지를 포함
    * 전체 프로젝트 또는 개별 파일의 이전 버전을 검색
    * 실험적으로 변경할 수 있는 분기(Branch)를 생성하면, 이는 기본 분기에 영향을 주지 않고 여러 사람이 동시에 다양한 작업을 수행할 수 있다. 향후 유지할 변경 내용을 기본 분기에서 병합(Marge)할 수 있음
    * 새 릴리즈를 표시를 위해 버전에 태그 부여

# CVS, SVN(Subersion), Perforce
이전 VSC 인스턴스는 중앙 집중식 서버를 사용하여 프로젝트 기록을 저장, 중앙 집중식 구성에서는 하나의 서버가 잠재적으로 단일 실패 지점일 가능성이 높기에 분상형인 `Git`으로 프로젝트의 전체 기록이 클라이언트와 서버에 모두 저장되어 네트워크에 연결하지 않고 파일을 편집하여 로컬로 체크 인할 수 있기에 서버가 다운되더라도 복사본이 존재한다.

## Git 용어
* 작업 트리 : 
    * 작업 중인 프로젝트를 포함하는 중첩된 디렉터리 및 파일 트리
* 리포지토리 : 
    * 작업 트리의 최상위 수준에 있는 디렉터리로, 프로젝트의 모든 기록 및 메타데이터를 여기에 보관한다. 작업 트리에 포함되지 않는 리포지토리로, 공유 및 백업에 사용되고 일반적으로 흔히 `.git`이라는 하나의 디렉터리다.
* 해시 : 
    * 해시 함수에서 생성된 숫자로, 파일 또는 다른 개체의 콘텐츠를 고정 자릿수로 나타낸다. Git은 160비트 길이의 해시를 사용하여, 결과를 이전 해시와 비교하여 파일이 변경되었는지 여부를 확인할 수 있다. 파일 시간 및 날짜 데이터 스탬프가 변경되었지만 파일 해시가 변경되지 않은 경우는 파일 내용이 변경되지 않았다는 것.
* 개체 : 
    * Git 리포지토리에는 각 SHA-1 해시를 통해 고유한 식별 유형이 포함된다. Blob 개체는 일반 파일을 포함하고, 트리 개체는 디렉터리를 뜻하며 이름과 해시 및 권한이 포함된다. Commit 개체는 특정 버전의 작업 트리를 나타내 Tag를 통해 커밋에 연결된 이름을 부여하는 개체이다.
* 커밋 :
    * 커밋에서 데이터베이스로 이름을 가져와 외부 사용자도 볼 수 있도록 변경 내용을 남기는 이것을 커밋이라 칭한다.
* 분기 :
    * 연결된 커밋의 명명된 시리즈로 분기의 가장 최근 커밋을 `헤드`라고 한다. 리포지토리를 초기화할 때 생성되는 기본 분기를 `main`이라 하여 혹은 `master`라고 칭하지만 현재는 대부분 `main`만 사용한다. 현재 분기의 헤드는 `HEAD`라고 뜻해 작업자가 분기에서 개별적으로 작업하고 나중에 변경 내용을 기본 분기에 병합할 수 있기 때문에 유용한 기능이다.
* 원격 :
    * 다른 Git 리포지토리에 관한 명명된 참조로 리포지토리를 만들 때 Git은 푸시 및 끌어오기 작업의 기본 원격인 `Origin`이라는 원격을 만든다.
* 명령어 / 옵션 :
    * `git push` 및 `git pull`과 같은 명령을 사용하여 수행한다. `git`은 주 명령이며 `push`혹은 `pull`은 하위 명령어로 Git내에서 작업을 수행하라는 명령을 지정한다. 일반적으로 명령은 `-`또는 `--`을 사용하는 옵션과 함께 사용한다.

## Git 명령줄 도구 <Git CommandLine tool>
GitHub Desktop을 포함하여 몇 가지의 GUI를 사용할 수 있다. 대부분의 IDE 프로그램에서 사용하지만 모든 Git의 기능을 구현하는 인터페이스는 없다.

## Git 과 GitHub
Git으로 주요 작업을 진행하면서 다른 참가자가 프로젝트에서 사용할 수 있는 DVCS()라고 칭하며 하나 이상의 로컬 분기로 작업 후 원격 리포지토리에 업로드하는 방법을 제공한다. Git과 GitHub의 클라우드 플랫품으로 프로젝트에서 협업하는 프로세스를 간소화하고 개발자와 사용자가 함께 작업하는 데 사용할 수 있는 웹 사이트로, 추가 명령줄 도구 및 흐름을 제공하여 원격 리포지토리 역할을 수행한다.

    * 토론
    * 끌어오기 요청
    * 알림
    * 레이블
    * 동작
    * 포크
    * 프로젝트
